DDS v2.0 · 제작 과정

동화 디자인 시스템은
이렇게 만들어졌습니다

색·글자 크기를 '이름'으로 관리하는 토큰을 정의하고, 상태별로 검증된 컴포넌트를 만들어,
design.md 한 문서로 정리·배포하기까지의 과정입니다.

자료 출처: Figma 디자인 파일 2026_DDS_v2_배포용 · 실제 화면 캡처
1

디자인 토큰 정의색·글자 크기를 코드값이 아닌 '이름'으로 관리

색 코드(#00AD50)나 숫자(16px)를 화면마다 직접 입력하는 대신, 의미가 담긴 이름으로 관리하도록 토큰 체계를 구축했습니다. 토큰은 원시 값(Primitive) → 의미 매핑(Semantic) → 부품 적용(Component)의 3단계로 구성됩니다.

색상 6단계 색상 체계

색은 용도에 따라 6개 그룹으로 나누어 정의했습니다. 브랜드 정체성을 담은 색부터, 실무에서 의미별로 가져다 쓰는 색까지 단계적으로 구성됩니다.

Brand Color · 브랜드 기본색 Primary · 대표색 Primitive Color · 원색 팔레트 Accent Color · 강조색 Semantic Color · 의미별 색 State Color · 상태색

Brand Brand Color — 동화기업 브랜드 컬러

동화기업의 정체성을 담은 그린 계열(Light Green·Green·Deep Green)과 Secondary 그레이를 각각 50~900단계로 정의했습니다.

Primitive Primitive Color — 기본 원색 팔레트

Blue·Cyan·Mint·Pink·Orange 등 색 계열별로 50~900까지 10단계 명도를 정의하여, 모든 화면이 같은 색 팔레트에서 색을 가져다 쓰도록 했습니다.

대표 색상 토큰 (실제 정의값)
Primary/green #00AD50 Accent/blue #006FF1 Accent/cyan #00B0B9 State/positive #00CE7D State/negative #FB4F4F

타이포 Font size — 글자 크기 단계

글자 크기는 font size/300(12px)부터 최대 56px까지 이름으로 단계화하여, 제목·본문·캡션의 위계가 모든 화면에서 일관되게 유지됩니다.

타이포 정의 완료 항목
✓ Font size 12 ~ 56px ✓ Font style — weight ✓ Font style — italic ✓ Letter spacing
예를 들어 '동화 그린'(#00AD50)이나 본문 크기(font size/400)를 바꾸고 싶을 때, 이름표 안의 값만 한 번 수정하면 그 이름을 쓴 모든 화면에 자동 반영됩니다. 디자이너·개발자가 같은 이름을 공유하므로 색·크기가 어긋날 일이 없습니다.
2

컴포넌트 제작자주 쓰는 UI를 상태별로 검증된 부품으로 마련

버튼·입력창·선택 메뉴·툴팁 등 자주 쓰는 UI를 재사용 가능한 부품으로 만들어 두었습니다. 각 부품은 기본·호버·선택·비활성·오류·로딩 등 상태별 모습까지 모두 정의되어 있어, 가져다 쓰기만 하면 누구나 동일한 형태와 동작을 구현할 수 있습니다.

Button 상태·스타일별 버튼

채움형·외곽선형·텍스트형 3가지 스타일에 대해 기본·강조·비활성·로딩 상태를 모두 정의했습니다.

Text field 입력 필드

기본·포커스·오류·비활성 등 상태별 입력창을 정의했습니다.

Select 선택 · 체크 · 라디오

드롭다운 선택 메뉴와 체크박스·라디오 버튼을 기본·선택·비활성 등 상태별로 정의했습니다.

Tooltip 설명 말풍선

본문만 있는 형태, 액션 버튼이 있는 형태, 닫기 버튼이 있는 형태와 함께 말풍선 꼬리의 방향까지 정의했습니다.

3

문서화 및 배포design.md 한 문서로 정리해 GitHub에 공개

위 토큰과 컴포넌트의 사용 규칙을 하나의 문서(design.md)로 정리하여 GitHub에 배포했습니다. 사람이 읽기 좋으면서 AI가 코드로 변환하기에도 좋은 형식으로 작성해, 누구나 열람·검색·활용하고 변경 이력까지 확인할 수 있습니다.

토큰·컴포넌트
Figma → 규칙 정리
design.md → 공개·공유
GitHub → 활용
전 직군 + AI
이제 디자이너가 아니더라도, AI와 함께 design.md의 기준만 따르면 누구나 동화 브랜드의 통일된 사용자 경험을 만들 수 있습니다.
